# 路径与文件线索收集工作流

## 固定首看文件

收到站点 URL 后，默认先看：
- `/robots.txt`
- `/sitemap.xml`
- `/favicon.ico`
- 首页 HTML
- 首页加载的 JS
- 常见后台入口：`/admin`、`/login`、`/manage`、`/dede`、`/system`

## robots.txt 的处理原则

`robots.txt` 通常不等于“高敏文件”，但它经常泄露：
- 后台路径
- 功能模块路径
- 安装/调试路径
- 上传目录
- CMS 组件目录
- 开发遗留目录

**规则：不要只读 robots.txt，要逐条跟进里面的路径。**

## 路径优先级

按下面顺序排查：
1. 用户明确给出的路径；
2. `robots.txt` / JS / 注释泄露路径；
3. 指纹产品的默认后台/模块路径；
4. 目录扫描新发现的路径。

## 遇到路径时怎么做

### 1. 先直接访问
看：
- 状态码
- 跳转
- 报错
- 参数
- 是否暴露版本、SQL 报错、调试信息

### 2. 识别它属于什么模块
例如：
- `/plus/`、`/member/`、`/dede/` 常见于 DedeCMS；
- `/install/` 可能是安装器；
- `/uploads/` 是上传目录；
- `/api/` 是接口面。

### 3. 立刻尝试最小验证
- 参数点：加 `'`、布尔差异、简单回显；
- 上传目录：直接访问、看是否执行脚本；
- 后台目录：尝试登录页、找验证码/弱口令/已知凭据；
- 调试页：看是否泄露环境、数据库、路径。

## 示例策略

如果 `robots.txt` 泄露 `/plus/recommend.php`：
1. 先访问原路径；
2. 看是否需要参数；
3. 对可疑参数做引号试探；
4. 一旦出现 SQL 报错或行为差异，立即切到 SQLi 流程；
5. 不要继续把时间花在更泛的目录爆破上。

## 首页 / JS 补充动作

- 用正则提取 `/api/...`、`/upload...`、`/admin...`、`debug`、`token`、`jsonp` 等关键词；
- 看是否有隐藏接口、资源根路径、上传回显路径；
- 看是否有 Webpack source map、调试变量、明文 key。
